// Collaborative Discussion 2 · Units 9–11 · Initial Post
Ontology Languages for the WWW — Why OWL2?

Dear Colleagues, following the definition of ontology given by Kalibatiene & Vasilecas (2011) as a "formal, explicit specification of a shared conceptualization", the topic of this second collaborative discussion is to identify which language is most useful to express ontologies for use by agents on the WWW.

In my personal opinion, the most useful language is OWL 2 (Web Ontology Language). Here are my key reasons:

🌐 Built for the Semantic Web
OWL2 was purposely created for the semantic web architecture — agents can discover, retrieve and reason over ontologies using standard web protocols, something KIF was never designed for. As Segaran et al. (2009) note, OWL is the current W3C standard for semantic web schemas.
🧠 Description Logic Foundation
OWL2 is based on Description Logics (DL), which guarantees that reasoning tasks will terminate. Other languages based on FOL cannot offer the same guarantee — a critical advantage for reliable agent behaviour.
🔧 Developer Flexibility
OWL2 offers three sublanguages — Existential Language (EL), Query Language (QL), and Rule Language (RL) — giving developers wide choice for their specific application needs, from lightweight mobile deployments to complex enterprise reasoning systems.

I look forward to hearing your opinions on this topic and discussing the merits of KIF, RDF and OWL-Lite in comparison.

— Andrea
