Initial Post

Dear Colleagues,

Following the definition of ontology given by Kalibatiene & Vasilecas (2011) as a ‚Äúformal, explicit specification of a shared conceptualization‚Äù, the topic of this second collaborative learning discussion is to identify which language is more useful to express ontologies to be used by agents.

In my personal opinion, the most useful language to express ontologies and to be used by agents in the WWW is the OWL 2 Web Ontology Language (shorten in OWL2 in this post). There are few reasons why I consider this language the most useful:

- OWL2 has been purposely created for the semantic web architecture so, in this case, the agents can easily discover, retrieve and reason over ontologies using the standard web protocols (one of the aspects where KIF was never designed for).

As highlighted by Segaran et al. (2009) ‚ÄúOWL is the current W3C standard for defining semantic web schemas, and tools and API support for OWL are rapidly expanding‚Äù.

- OWL2 is based on DL (Description Logics) and that is guarantying that the reasoning tasks will be terminated. Other languages based on FOL cannot guarantee the same.

- OWL2 offers more flexibility to developers because it provides: Existential Language, Query Language and Rule Language which give developers wide choice for their applications.

I would like to hear your opinions about this topic.

Andrea
