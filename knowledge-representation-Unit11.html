// Collaborative Discussion 2 · Units 9–11 · Summary Post
Ontology Languages for the WWW — Discussion Summary

Dear Colleagues, this discussion forum has been genuinely enriching, providing valuable insights into the practical considerations of ontology languages for software agents on the WWW.

In my initial post (Trevisi, 2026), I argued that OWL 2 is the most useful language for expressing ontologies for software agents, citing three main reasons: its purpose-built design for semantic web architecture, its Description Logic foundation ensuring decidable reasoning, and its flexibility through multiple sub-languages.

Lauretta Oghenevurie raised important practical deployment challenges regarding OWL 2's computational complexity in large-scale systems, introducing OWL 2 profiles as a solution to balance expressiveness with performance. This highlighted that OWL 2's value lies not just in theoretical capabilities, but in its modular flexibility within broader semantic stacks (Krötzsch, 2012).

Matthew Bullen brought a crucial practical dimension — framing ontology implementation as a data integration challenge rather than purely semantic description. The discussion of OBDA methods and R2RML mapping demonstrated how OWL 2 serves as a foundational semantic layer while complementary technologies handle data transformation and schema alignment (Poggi et al., 2008).

These discussions reinforced that OWL 2's strength is its adaptability — functioning as a configurable framework that provides standardised ontological foundations while allowing specialised tools to address specific challenges. A really important debate that enriched my understanding of ontology languages significantly.

— Andrea
